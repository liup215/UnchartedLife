# Project Brief: Godot ARPG

## Core Request
The user wants to develop an Action RPG (ARPG) using the Godot game engine. The primary focus is on creating a robust and scalable architecture with an emphasis on strong code organization and high reusability.

## Key Goals
- **Game Genre:** Action RPG (ARPG)
- **Engine:** Godot
- **Architectural Requirements:**
    - **Scalable:** The architecture should support the addition of new features, content, and systems without requiring major refactoring.
    - **Organized:** The project structure (files, scenes, scripts) should be clean, logical, and easy for developers to navigate.
    - **Reusable:** Components, systems, and assets should be designed for maximum reusability to accelerate development and reduce redundancy.

## Initial Scope
The initial task is to provide architectural design recommendations and best practices for building a Godot-based ARPG that meets the above requirements. This involves suggesting patterns for directory structure, scene composition, code organization, and the design of core game systems.
